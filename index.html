Experiment 1: College Course Registration System (Week 1)

Scenario

A college wants to maintain records of students and the courses they enroll in.

Tasks to Perform:
• Create a table Student with appropriate attributes (StudentID, Name, Age, Email).
• Create a table Course (CourseID, CourseName, Credits).
• Create a table Enrollment (EnrollmentID, StudentID, CourseID).
• Set PRIMARY KEY for all tables and Apply NOT NULL on required fields.
• Ensure Email is UNIQUE.
• Apply CHECK constraint on Age (should be ≥ 18).
• Apply FOREIGN KEY constraints in Enrollment table.
• Insert at least 3 records in each table.
• Add a new column PhoneNumber to Student table using ALTER.
• Modify the Credits column to ensure values are between 1 and 5.
• Rename table Course to Subject.
• Remove all records from Enrollment table using TRUNCATE.
• Drop the Subject table.

SQL Code

-- Create Tables
CREATE TABLE Student (
    StudentID INT PRIMARY KEY,
    Name VARCHAR(50) NOT NULL,
    Age INT CHECK (Age >= 18),
    Email VARCHAR(100) UNIQUE NOT NULL
);

CREATE TABLE Course (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(50) NOT NULL,
    Credits INT NOT NULL
);

CREATE TABLE Enrollment (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    FOREIGN KEY (StudentID) REFERENCES Student(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Course(CourseID)
);

-- Insert Data

(3, 'Amit', 21, 'amit@gmail.com');

(103, 'Python', 5);

INSERT INTO Enrollment VALUES 
(1, 1, 101),
(2, 2, 102),
(3, 3, 103);

-- Alter Tables
ALTER TABLE Student ADD PhoneNumber VARCHAR(15);
ALTER TABLE Course ADD CONSTRAINT chk_credits 
    CHECK (Credits BETWEEN 1 AND 5);
ALTER TABLE Course RENAME TO Subject;

-- Delete Operations
TRUNCATE TABLE Enrollment;
DROP TABLE Enrollment;
DROP TABLE Subject;

Experiment 3: Bank Account Management System (Week 3)

Scenario

A bank wants to manage customer accounts and transactions securely. The database stores customer account details and balances. The system should support data updates, controlled access, and safe transaction handling.

Tasks to Perform:
• Create a table Account with attributes: (AccountID, CustomerName, Balance, AccountType).
• Insert at least 5 records.
• Increase balance by 10% for all customers with AccountType = 'Savings'.
• Update AccountType to 'Premium' where Balance > 50,000.
• Delete accounts where Balance is less than 1000.
• Use arithmetic operator to display Balance after adding ₹500.
• Use logical operators to display customers with Balance between 10,000 and 50,000.
• Use pattern matching to find customers whose names start with 'A'.
• Use at least one string function (e.g., convert names to uppercase).
• Create a role BankClerk.
• Grant permission to: SELECT and INSERT on Account table.
• Revoke INSERT permission from BankClerk.
• Start a transaction, transfer ₹5000 from one account to another using UPDATE.
• Create a SAVEPOINT and perform another update (e.g., deduct ₹200 as charges).
• Rollback to the savepoint and Commit the transaction.

SQL Code

-- Create Table
CREATE TABLE Account (
    AccountID INT PRIMARY KEY,
    CustomerName VARCHAR(50) NOT NULL,
    Balance DECIMAL(10,2) NOT NULL,
    AccountType VARCHAR(20) NOT NULL
);

-- Insert Data
INSERT INTO Account VALUES 
(1, 'Amit', 20000, 'Savings'),
(2, 'Rahul', 60000, 'Current'),
(3, 'Anjali', 8000, 'Savings'),
(4, 'Sneha', 75000, 'Savings'),
(5, 'Arjun', 500, 'Current');

-- Update Operations
UPDATE Account SET Balance = Balance + (Balance * 0.10) 
    WHERE AccountType = 'Savings';
UPDATE Account SET AccountType = 'Premium' 
    WHERE Balance > 50000;
DELETE FROM Account WHERE Balance < 1000;

-- Select Queries
SELECT CustomerName, Balance + 500 FROM Account;
SELECT * FROM Account 
    WHERE Balance BETWEEN 10000 AND 50000;
SELECT * FROM Account 
    WHERE CustomerName LIKE 'A%';

-- Role Management
CREATE ROLE BankClerk;
GRANT SELECT, INSERT ON Account TO BankClerk;
REVOKE INSERT ON Account FROM BankClerk;

-- Transaction with Savepoint
START TRANSACTION;
UPDATE Account SET Balance = Balance - 5000 
    WHERE AccountID = 2;
UPDATE Account SET Balance = Balance + 5000 
    WHERE AccountID = 1;
SAVEPOINT sp1;
UPDATE Account SET Balance = Balance - 200;
ROLLBACK TO sp1;
COMMIT;

• Create a table Sales with attributes: (SaleID, ProductName, Category, Quantity, Price).
• Display the total quantity sold for each category.
• Display the average price of products in each category.
• Find the maximum and minimum price in each category.
• Display total sales amount (Quantity × Price) for each category.
• Count the number of products in each category.
• Display categories where total sales amount is greater than 5000.

• Display the category with the highest total sales.

SQL Code

-- Create Table

    SaleID INT PRIMARY KEY,
    ProductName VARCHAR(50) NOT NULL,
    Category VARCHAR(50) NOT NULL,
    Quantity INT NOT NULL,
    Price DECIMAL(10,2) NOT NULL
);

-- Insert Data

(2, 'Mobile', 'Electronics', 5, 20000),

(4, 'Table', 'Furniture', 3, 4000),
(5, 'Pen', 'Stationery', 50, 20),
(6, 'Notebook', 'Stationery', 30, 100);

-- Aggregate Queries
SELECT Category, SUM(Quantity) FROM Sales GROUP BY Category;
SELECT Category, AVG(Price) FROM Sales GROUP BY Category;
SELECT Category, MAX(Price), MIN(Price) 
    FROM Sales GROUP BY Category;
SELECT Category, SUM(Quantity * Price) 
    FROM Sales GROUP BY Category;
SELECT Category, COUNT(*) FROM Sales GROUP BY Category;

-- GROUP BY with HAVING
SELECT Category, SUM(Quantity * Price) 
    FROM Sales GROUP BY Category 
    HAVING SUM(Quantity * Price) > 5000;
SELECT Category, AVG(Price) 
    FROM Sales GROUP BY Category 
    HAVING AVG(Price) > 1000;

-- Highest Selling Category
SELECT Category, SUM(Quantity * Price) 
    FROM Sales GROUP BY Category 
    ORDER BY SUM(Quantity * Price) DESC LIMIT 1;

Experiment 6: University Examination System (Week 6)

Scenario

A university maintains records of students, subjects, and their marks. The system needs to generate meaningful reports by combining data from multiple tables and also create reusable views.

Tasks to Perform:
• Create the following tables:
    - Student (StudentID, StudentName, Department)
    - Subject (SubjectID, SubjectName)
    - Marks (StudentID, SubjectID, MarksObtained)
• Insert at least 5–6 records in each table.
• Display Student Name and Subject Name with Marks using INNER JOIN.
• Display all students along with their marks (include students even if they have no marks) using LEFT JOIN.
• Display all subjects along with student details (include subjects even if no student has taken them) using RIGHT JOIN.
• Display all possible combinations of students and subjects using CROSS JOIN.
• Display the department with the highest total marks.

SQL Code

-- Create Tables
CREATE TABLE Student (
    StudentID INT PRIMARY KEY,
    StudentName VARCHAR(50) NOT NULL,
    Department VARCHAR(50) NOT NULL
);

CREATE TABLE Subject (
    SubjectID INT PRIMARY KEY,
    SubjectName VARCHAR(50) NOT NULL
);

CREATE TABLE Marks (
    StudentID INT,
    SubjectID INT,
    MarksObtained INT,
    FOREIGN KEY (StudentID) REFERENCES Student(StudentID),
    FOREIGN KEY (SubjectID) REFERENCES Subject(SubjectID)
);

-- Insert Data
INSERT INTO Student VALUES 
(1, 'Rahul', 'Computer'),
(2, 'Sneha', 'Mechanical'),
(3, 'Amit', 'Civil'),
(4, 'Anjali', 'Computer'),
(5, 'Rohan', 'Electrical');

INSERT INTO Subject VALUES 
(101, 'DBMS'),
(102, 'Mathematics'),
(103, 'Physics'),
(104, 'Java'),
(105, 'Mechanics');

INSERT INTO Marks VALUES 
(1, 101, 85), (1, 104, 90), (2, 105, 78),
(3, 102, 88), (4, 101, 92), (5, 103, 75);

-- INNER JOIN
SELECT Student.StudentName, Subject.SubjectName, 
    Marks.MarksObtained 
    FROM Marks 
    INNER JOIN Student ON Marks.StudentID = Student.StudentID 
    INNER JOIN Subject ON Marks.SubjectID = Subject.SubjectID;

-- LEFT JOIN
SELECT Student.StudentName, Subject.SubjectName, 
    Marks.MarksObtained 
    FROM Student 
    LEFT JOIN Marks ON Student.StudentID = Marks.StudentID 
    LEFT JOIN Subject ON Marks.SubjectID = Subject.SubjectID;

-- CROSS JOIN
SELECT Student.StudentName, Subject.SubjectName 
    FROM Student CROSS JOIN Subject;

-- Highest Scoring Department
SELECT Student.Department, SUM(Marks.MarksObtained) 
    FROM Student 
    INNER JOIN Marks ON Student.StudentID = Marks.StudentID 
    GROUP BY Student.Department 
    ORDER BY SUM(Marks.MarksObtained) DESC LIMIT 1;

    - Display employees whose salary is greater than the average salary of all employees.

Experiment 10: Library Management System using Cursor (Week 10)

Scenario

A college library maintains records of books and the students who issue them. The librarian wants to calculate the total number of books issued and display the details of each issued book using a cursor inside a stored procedure. This will help in generating reports for issued books.

Tasks to Perform:
• Create the following tables:
    - Books (BookID [Primary Key], BookTitle, Author, AvailableCopies)
    - IssuedBooks (IssueID [Primary Key], StudentName, IssueDate, BookID [Foreign Key])
• Insert at least 5 records into each table.
• Create a Stored Procedure using Cursor that:
    - Retrieves all issued books from the IssuedBooks table.
    - Displays BookID, StudentName, and IssueDate one by one using a cursor.
• Execute the stored procedure to display the issued book details.

SQL Code

-- Create Tables
CREATE TABLE Books (
    BookID INT PRIMARY KEY,
    BookTitle VARCHAR(100) NOT NULL,
    Author VARCHAR(50) NOT NULL,
    AvailableCopies INT NOT NULL
);

CREATE TABLE IssuedBooks (
    IssueID INT PRIMARY KEY,
    StudentName VARCHAR(50) NOT NULL,
    IssueDate DATE NOT NULL,
    BookID INT,
    FOREIGN KEY (BookID) REFERENCES Books(BookID)
);

-- Insert Data
INSERT INTO Books VALUES 
(101, 'DBMS Concepts', 'Korth', 5),
(102, 'Java Programming', 'Herbert Schildt', 3),
(103, 'Python Basics', 'Guido Rossum', 4),
(104, 'Computer Networks', 'Forouzan', 2),
(105, 'Operating System', 'Galvin', 6);

INSERT INTO IssuedBooks VALUES 
(1, 'Rahul', '2026-05-01', 101),
(2, 'Sneha', '2026-05-02', 102),
(3, 'Amit', '2026-05-03', 103),
(4, 'Anjali', '2026-05-04', 104),
(5, 'Rohan', '2026-05-05', 105);

-- Create Stored Procedure with Cursor
DELIMITER $$
CREATE PROCEDURE DisplayIssuedBooks()
BEGIN
    DECLARE done INT DEFAULT 0;
    DECLARE b_id INT;
    DECLARE s_name VARCHAR(50);
    DECLARE i_date DATE;
    
    DECLARE book_cursor CURSOR FOR 
        SELECT BookID, StudentName, IssueDate 
        FROM IssuedBooks;
    
    DECLARE CONTINUE HANDLER FOR NOT FOUND 
        SET done = 1;
    
    OPEN book_cursor;
    read_loop: LOOP
        FETCH book_cursor INTO b_id, s_name, i_date;
        IF done = 1 THEN LEAVE read_loop; END IF;
        SELECT b_id AS BookID, s_name AS StudentName, 
               i_date AS IssueDate;
    END LOOP;
    CLOSE book_cursor;
END $$
DELIMITER ;

-- Execute Procedure
CALL DisplayIssuedBooks();

Tasks to Perform:
• Create the following tables:
    - Products (ProductID [Primary Key], ProductName, Price)
    - Orders (OrderID [Primary Key], Quantity, OrderDate, ProductID [Foreign Key])
• Insert at least 5 records into each table.
• Create a Function that returns the total revenue generated from all orders.
• Execute the function to display the total revenue.

SQL Code

-- Create Tables
CREATE TABLE Products (
    ProductID INT PRIMARY KEY,
    ProductName VARCHAR(100) NOT NULL,
    Price DECIMAL(10,2) NOT NULL
);

CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    Quantity INT NOT NULL,
    OrderDate DATE NOT NULL,
    ProductID INT,
    FOREIGN KEY (ProductID) REFERENCES Products(ProductID)
);

-- Insert Data
INSERT INTO Products VALUES 
(101, 'Laptop', 50000),
(102, 'Mobile', 20000),
(103, 'Headphones', 3000),
(104, 'Keyboard', 1500),
(105, 'Mouse', 800);

INSERT INTO Orders VALUES 
(1, 2, '2026-05-01', 101),
(2, 5, '2026-05-02', 102),
(3, 3, '2026-05-03', 103),
(4, 4, '2026-05-04', 104),
(5, 6, '2026-05-05', 105);

-- Create Function to Calculate Total Revenue
DELIMITER $$
CREATE FUNCTION TotalRevenue() 
    RETURNS DECIMAL(15,2) DETERMINISTIC
BEGIN
    DECLARE total DECIMAL(15,2);
    SELECT SUM(Orders.Quantity * Products.Price) 
        INTO total 
        FROM Orders 
        INNER JOIN Products 
        ON Orders.ProductID = Products.ProductID;
    RETURN total;
END $$
DELIMITER ;

-- Execute Function
SELECT TotalRevenue() AS Total_Revenue;

SQL Code

CREATE TABLE Accounts (
    AccountID INT PRIMARY KEY,
    CustomerName VARCHAR(50) NOT NULL,
    Balance DECIMAL(10,2) NOT NULL
);

CREATE TABLE Transactions (
    TransactionID INT PRIMARY KEY,
    TransactionType VARCHAR(20) NOT NULL,
    Amount DECIMAL(10,2) NOT NULL,
    AccountID INT,
    FOREIGN KEY (AccountID) REFERENCES Accounts(AccountID)
);

-- Insert Data
INSERT INTO Accounts VALUES 
(101, 'Rahul', 10000),
(102, 'Sneha', 15000),
(103, 'Amit', 8000),
(104, 'Anjali', 20000),
(105, 'Rohan', 5000);

INSERT INTO Transactions VALUES 
(1, 'Deposit', 2000, 101),
(2, 'Withdrawal', 3000, 102),
(3, 'Deposit', 1500, 103),
(4, 'Withdrawal', 4000, 104),
(5, 'Deposit', 1000, 105);

-- Create BEFORE INSERT Trigger
DELIMITER $$
CREATE TRIGGER Check_Balance 
    BEFORE INSERT ON Transactions 
    FOR EACH ROW 
BEGIN
    DECLARE current_balance DECIMAL(10,2);
    SELECT Balance INTO current_balance 
        FROM Accounts 
        WHERE AccountID = NEW.AccountID;
    
    IF NEW.TransactionType = 'Withdrawal' 
        AND NEW.Amount > current_balance THEN
        SIGNAL SQLSTATE '45000' 
        SET MESSAGE_TEXT = 'Insufficient Balance';
    ELSEIF NEW.TransactionType = 'Withdrawal' THEN
        UPDATE Accounts SET Balance = Balance - NEW.Amount 
            WHERE AccountID = NEW.AccountID;
    ELSEIF NEW.TransactionType = 'Deposit' THEN
        UPDATE Accounts SET Balance = Balance + NEW.Amount 
            WHERE AccountID = NEW.AccountID;
    END IF;
END $$
DELIMITER ;

-- Test Trigger with Valid Withdrawal
INSERT INTO Transactions 
    VALUES (6, 'Withdrawal', 2000, 101);

-- Display Updated Balances
SELECT * FROM Accounts;

• Create a trigger that:

• Test the trigger by attempting to insert invalid marks.
• Display the valid records stored in the table.

-- Create Table
CREATE TABLE StudentMarks (

    Subject VARCHAR(50) NOT NULL,
    Marks INT NOT NULL
);

    BEFORE INSERT ON StudentMarks 
    FOR EACH ROW 
BEGIN
    IF NEW.Marks < 0 OR NEW.Marks > 100 THEN
        SIGNAL SQLSTATE '45000' 
        SET MESSAGE_TEXT = 
            'Marks should be between 0 and 100';

    BEFORE UPDATE ON StudentMarks 
    FOR EACH ROW

    IF NEW.Marks < 0 OR NEW.Marks > 100 THEN

        SET MESSAGE_TEXT = 
            'Marks should be between 0 and 100';
    END IF;
END $$
DELIMITER ;

    SET Marks = -10

Experiment 14: MongoDB CRUD Operations (Week 14)

Scenario

A college wants to manage student course registrations using MongoDB. Each student registers for different courses, and the system needs to store and manage student information in a database. The administrator should be able to add new student records, view student details, update course information, and delete records when necessary. MongoDB CRUD operations will be used to perform these tasks.

Tasks to Perform:
• Create a MongoDB database named CollegeDB.
• Create a collection named Students.
• Insert student records using the save() method with the following fields: StudentID, Name, Course, Year, Marks. Insert at least 5 student documents into the collection.
• Read Operations:
    - Display all student records from the collection.
    - Display students who belong to the "Computer Engineering" course.
    - Display students who have marks greater than 70.
• Update Operation: Update the course of a student whose StudentID is 102.
• Delete Operation: Delete the record of a student whose StudentID is 105.
• Use Logical Operators:
    - Display students who have marks greater than 60 AND belong to the "IT" course using $and.
    - Display students who belong to "Computer Engineering" OR "IT" courses using $or.
